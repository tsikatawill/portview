Overview

Portview helps developers answer one annoying question quickly: “What is using this port?” Instead of juggling lsof, netstat, or platform-specific kill commands, the app gives you a visual list of active ports, process names, PIDs, ownership details, and connection state.

Core features

  • Scan active ports and bound processes
  • Pin common development ports like 3000 and 5173
  • Filter by state and search by port, process, or PID
  • Kill or force-kill problematic processes

Best for

  • Frontend and backend local development
  • Recovering from stuck dev servers
  • Tracking port ownership across multiple projects
  • Quick desktop troubleshooting without terminal memorization

Installation

In the repository root, install dependencies and run the app in development mode: 

npm install
npm run dev

To build the application bundles without starting the desktop app: 

npm run build

For project verification during development:

npm run typecheck

How to use Portview

  1. Launch the app and wait for the initial port scan to complete.
  2. Use the search bar to find a port, PID, or process by name.
  3. Pin ports you care about so they stay easy to monitor.
  4. Use the refresh controls for manual or automatic rescans.
  5. Kill a process when you are sure it is safe to terminate.
Understand the states Settings guidance

Port states

Portview uses clearer labels than raw TCP state names so you can tell what matters at a glance.

In Use

The port is actively bound and waiting for incoming connections. This is the state that usually blocks a dev server from starting on the same port.

Connected

There is an active connection on this socket right now. Useful for diagnostics, though not always the thing causing a startup conflict.

Close Wait

The remote side has disconnected, but the local process has not fully finished cleaning up the connection yet.

Closed

The socket is no longer open. This is generally less important for conflict resolution, but still useful context when investigating network behavior.

Settings and theme

The navbar now includes a settings icon for quick access to the app preferences screen. From there, users can:

  • Switch between Dark, Light, and System themes
  • Toggle confirmation before killing a process
  • Enable or disable auto refresh
  • Adjust the live scan interval

Dark mode is now the default for new users. Existing users will keep whichever theme has already been persisted in their local app settings unless they change it manually.

Troubleshooting

The port list looks empty

Try a manual refresh, clear active filters, and verify the target service is actually running locally.

A port will not free up

Check whether the process immediately restarts after being killed. Watchers, background daemons, or another parent process may be bringing it back.

The wrong theme is showing

Open the settings page from the navbar and explicitly set the theme to Dark, Light, or System to override the default.
